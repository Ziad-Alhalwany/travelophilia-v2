# CRM Required Fields — Travelophilia

> الهدف: كل مرحلة لها “حد أدنى” من البيانات يمنع الفوضى ويخلي التسعير/التشغيل صح.

---

## 1) حقول مشتركة (Core)
- Lead ID (أو رقم)
- الاسم
- رقم واتساب
- مصدر الـLead (IG/FB/SEO/Referral/Direct)
- الخدمة: (Template Trip / Custom Trip / Flight / Visa / Support)
- آخر تواصل (تاريخ)
- الحالة (Stage)

---

## 2) حقول حسب نوع الخدمة

### A) Custom Trip
- الوجهة/المدن
- تواريخ (مرن/ثابت)
- عدد الأفراد + أعمار (لو أطفال)
- Budget range (تقريبي)
- نوع الإقامة (3/4/5 نجوم - شاليه - شقة)
- تفضيلات: (بحر/مغامرة/تاريخ/استجمام)
- حساسية: (privacy / luxury / group)

### B) Template Trip (باكدج جاهز)
- اسم الرحلة/الكود
- التاريخ المحتمل
- عدد الأفراد
- Upgrade options (غرفة/نقل/أنشطة)

### C) Flight / Visa
- جنسية
- وجهة
- تواريخ
- عدد المسافرين
- بيانات جواز (لاحقًا، بعد موافقة العميل)

---

## 3) حقول مالية (Finance Touchpoints)
- سعر العرض (Option A/B)
- العربون المطلوب + موعده
- سياسة إلغاء مختصرة أُبلغ بها العميل (Yes/No)
- Discounts applied (إن وُجدت) + سبب + موافقة

---

## 4) حقول تشغيل (Ops Handoff)
- نقطة تجمع/مدينة انطلاق
- وسيلة تواصل بديلة
- ملاحظات صحية/حساسية/طلبات خاصة
- أسماء المسافرين (قبل التأكيد النهائي)
